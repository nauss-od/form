## 1. نظرة عامة على المشروع

منصة **التدريب الخارجي لجامعة نايف** هي تطبيق ويب متكامل طُوِّر خصيصاً لتلبية احتياجات **وكالة الجامعة للتدريب**، وتمتد خدماتها لتشمل إدارات متعددة في الجامعة. تُتيح المنصة إدارة دورات التدريب الخارجية من بداية إنشاء الدورة حتى إصدار التأمين الطبي الدولي لجميع المشاركين، وذلك عبر تدفق عمل رقمي متكامل يُلغي الحاجة للورق بالكامل.

### المشكلة التي تحلّها المنصة

قبل هذه المنصة، كانت وكالة التدريب تعتمد على النماذج الورقية لجمع بيانات المشاركين في الدورات الخارجية لأغراض إصدار التأمين الطبي الدولي، مما كان يؤدي إلى:

- ضياع البيانات وصعوبة الأرشفة
- إهدار وقت الموظفين في إعادة إدخال البيانات يدوياً
- صعوبة إصدار قوائم المشاركين الرسمية بالشكل المطلوب
- عدم وجود سجل تدقيق يثبت تاريخ الإجراءات

### الحل الذي توفّره المنصة

- نموذج إلكتروني يملؤه المشاركون مباشرة عبر رابط آمن فريد لكل دورة
- استخراج تلقائي للبيانات من صورة جواز السفر (OCR داخل المتصفح — دون إرسال الصورة لأي خادم خارجي)
- توليد قائمة مشاركين PDF رسمية واحترافية بضغطة زر واحدة
- أرشفة كاملة مع سجل تدقيق لكل عملية
- إصدار التأمين وحذف الملفات الحساسة بشكل آلي بعد الإصدار

## 2. الأهداف والفائدة التنظيمية

| الهدف | التفاصيل |
|---|---|
| **رقمنة العمليات** | إلغاء النماذج الورقية في جمع بيانات المشاركين |
| **توفير الوقت** | تقليل وقت جمع بيانات 30 مشاركاً من يومين إلى ساعات |
| **الدقة** | OCR تلقائي من جواز السفر يقلل أخطاء الإدخال اليدوي |
| **الاحترافية** | قائمة PDF رسمية تحمل شعار الجامعة وبيانات الدورة |
| **الأمن والخصوصية** | تشفير البيانات + سجل تدقيق + معالجة OCR محلية |
| **التوسّع** | قابل للتوسع ليخدم إدارات الجامعة المختلفة |

### الإدارات المستفيدة

- **وكالة الجامعة للتدريب** — المستفيد الأول والمشغّل الرئيسي
- الإدارات الأكاديمية المرتبطة بالتدريب الخارجي
- إدارة الموارد البشرية (بيانات الموظفين المشاركين)
- الإدارة المالية (بيانات IBAN لأغراض البدلات المالية)

## 5. قاعدة البيانات وهيكل البيانات

المنصة تستخدم **PostgreSQL** مع **Prisma ORM**. الجداول تُنشأ تلقائياً من خلال `prisma db push` دون الحاجة لكتابة SQL يدوياً.

### جدول `User` — المستخدمون

```
الحقل          النوع       الوصف
─────────────────────────────────────────────────────
id             String      معرّف فريد (CUID)
name           String      الاسم الكامل
email          String      البريد الإلكتروني (فريد)
passwordHash   String      كلمة المرور (bcrypt — لا تُعرَض أبداً)
mobile         String?     رقم الجوال (اختياري)
extension      String?     الرقم الداخلي (اختياري)
role           Enum        MANAGER | EMPLOYEE
isActive       Boolean     مفعّل / موقوف
lastLoginAt    DateTime?   آخر تسجيل دخول
createdAt      DateTime    تاريخ الإنشاء
```

### جدول `Course` — الدورات

```
الحقل              النوع       الوصف
─────────────────────────────────────────────────────
id                 String      معرّف فريد (CUID)
createdByUserId    String      مرجع للمستخدم المنشئ
activityName       String?     اسم النشاط/البرنامج
venue              String?     مكان الانعقاد
startDate          DateTime?   تاريخ البدء
endDate            DateTime?   تاريخ الانتهاء
participantCount   Int?        العدد المتوقع
approvalFileUrl    String?     رابط ملف الموافقة الإدارية
approvalFileName   String?     اسم ملف الموافقة
publicToken        String      رمز فريد لرابط النموذج العام
status             Enum        PUBLISHED | CLOSED
insuranceIssuedAt  DateTime?   تاريخ إصدار التأمين
```

### جدول `CourseStaff` — مرشحو جامعة نايف

```
الحقل       النوع      الوصف
──────────────────────────────────────────
id          String     معرّف فريد
courseId    String     مرجع للدورة
name        String     الاسم بالإنجليزي
passportNo  String?    رقم الجواز
mobile      String?    رقم الجوال
jobTitle    String     المسمى الوظيفي (من القائمة المحددة)
sortOrder   Int        ترتيب الظهور في القائمة
```

**المسميات الوظيفية المتاحة:**

| المسمى | الدور |
|---|---|
| Scientific Supervisor | المشرف العلمي |
| Trainer 1 | المدرب الأول |
| Trainer 2 | المدرب الثاني |
| Translator | المترجم |
| Coordinator | المنسق |
| Operations Manager | مدير العمليات |

### جدول `Submission` — بيانات المشاركين

```
الحقل              النوع      الوصف
──────────────────────────────────────────────────────
id                 String     معرّف فريد
courseId           String     مرجع للدورة
fullNamePassport   String     الاسم الكامل حسب الجواز
passportNumber     String     رقم جواز السفر
passportExpiry     DateTime   تاريخ انتهاء الجواز
nationalId         String     رقم الهوية الوطنية (10 أرقام)
mobile             String     رقم الجوال (+966XXXXXXXXX)
birthDate          DateTime   تاريخ الميلاد
iban               String     رقم الآيبان البنكي (SA + 22 رقم)
```

### جدول `SubmissionFile` — ملفات المرفقات

```
الحقل        النوع      الوصف
──────────────────────────────────────────────────────
id           String     معرّف فريد
submissionId String     مرجع للتسجيل
fileType     Enum       PASSPORT | NATIONAL_ID
fileName     String     اسم الملف
fileSize     Int?       الحجم بالبايت
fileData     Bytes?     بيانات الملف (Binary مخزّن في DB)
mimeType     String?    نوع MIME للملف
```

> **ملاحظة تقنية مهمة:** الملفات مخزّنة كـ Binary مباشرة داخل قاعدة البيانات PostgreSQL. يُبسِّط هذا النسخ الاحتياطي ويلغي الحاجة لتهيئة خادم ملفات منفصل (S3, MinIO...)، لكنه يزيد حجم قاعدة البيانات. عند تجاوز 10,000 مشارك تاريخياً، يُوصى بتقييم الانتقال إلى تخزين خارجي. **الملفات تُحذَف نهائياً بعد إصدار التأمين لكل دورة.**

### جدول `AuditLog` — سجل التدقيق

```
الحقل       النوع      الوصف
──────────────────────────────────────────────────────
id          String     معرّف فريد
userId      String?    المستخدم المنفّذ (null إذا حُذف الحساب)
action      String     الإجراء المنفّذ
entityType  String     نوع الكيان (Course, Submission, User...)
entityId    String     معرّف الكيان المتأثر
metaJson    Json?      تفاصيل إضافية كـ JSON
createdAt   DateTime   الطابع الزمني الدقيق
```

### مخطط العلاقات

```
User ─[1:N]─► Course ─[1:N]─► Submission ─[1:N]─► SubmissionFile
                  └──[1:N]──► CourseStaff

User ─[1:N]─► AuditLog
```

## 7. أدوار المستخدمين والصلاحيات

### `MANAGER` — المدير

- رؤية جميع دورات المنصة لجميع الموظفين
- الوصول الكامل للوحة الإدارة: المستخدمون، سجل التدقيق، التقارير
- تصدير قوائم PDF لأي دورة
- إمكانية التبديل لعرض "كموظف" لرؤية دوراته الخاصة فقط

### `EMPLOYEE` — الموظف

- رؤية دوراته التي أنشأها فقط
- إنشاء دورات جديدة وإدارتها
- إضافة وتعديل وحذف مرشحي جامعة نايف لدوراته
- إصدار التأمين لدوراته
- تصدير قائمة المشاركين PDF

### المشارك (زائر — بدون حساب)

- الوصول لنموذج التسجيل عبر الرابط الخاص بالدورة فقط
- رفع صور الجواز والهوية
- استخدام ميزة مسح جواز السفر (OCR داخل المتصفح)

> **مبدأ التصميم:** النماذج العامة وخدمة الملفات **مفتوحة بتصميم متعمد ومدروس** — المشاركون يُسجِّلون بياناتهم دون حساب. لا يجوز تعديل هذا المبدأ لأنه جوهر طريقة عمل المنصة.

## 10. الأمن والحماية

### طبقات الحماية المُطبَّقة

#### ١. ترويسات HTTP الأمنية (Security Headers)
```
X-Content-Type-Options:    nosniff           (يمنع sniffing نوع المحتوى)
X-Frame-Options:           DENY              (يمنع تضمين الصفحات في iframes)
X-XSS-Protection:          1; mode=block     (حماية XSS إضافية)
Referrer-Policy:           strict-origin-when-cross-origin
Permissions-Policy:        camera=(), microphone=(), geolocation=()
Cache-Control (API):       private, no-store (منع التخزين المؤقت للبيانات)
```

#### ٢. المصادقة والجلسات
- كلمات المرور مشفّرة بـ **bcrypt** مع salt تلقائي — لا تُخزَّن في نص صريح أبداً
- الجلسات عبر **JWT** محفوظ في **httpOnly Cookie** — مقاوم لهجمات XSS
- مدة الجلسة: **7 أيام** مع تجديد تلقائي عند كل طلب

#### ٣. تحديد معدل الطلبات (Rate Limiting)
- تسجيل الدخول: **10 محاولات فاشلة** كحد أقصى لكل IP في 15 دقيقة
- بعد تجاوز الحد: حظر مؤقت مع رسالة عربية واضحة للمستخدم

#### ٤. التحقق من المدخلات (Input Validation)
- التحقق من صيغة رقم الجواز والهوية والآيبان والجوال
- قائمة بيضاء للمسميات الوظيفية المسموح بها (Allowlist)
- رفض الملفات غير الصور في نماذج المرفقات
- حجم أقصى لكل ملف: 50MB

#### ٥. عزل البيانات بين المستخدمين
- الموظف لا يرى إلا دوراته التي أنشأها هو
- المدير يرى الكل، لكنه يستطيع التبديل لعرض "كموظف"
- لا يوجد أي تسريب لبيانات مستخدم لمستخدم آخر في الـ APIs

#### ٦. سجل التدقيق الكامل
- كل إجراء حساس يُسجَّل مع: الوقت الدقيق، هوية المستخدم، التفاصيل
- لا يمكن حذف أو تعديل سجلات التدقيق من الواجهة

#### ٧. خصوصية بيانات المشاركين
- صور الجوازات: تُعالَج OCR محلياً في متصفح المشارك — **لا تُرسَل لأي خادم خارجي**
- الملفات (صور الجوازات والهويات) تُحذَف نهائياً من قاعدة البيانات فور إصدار التأمين
- رسالة الخصوصية تظهر للمشارك في بداية النموذج

## 11. الميزات والوظائف التفصيلية

### أ) إدارة الدورات

- إنشاء دورة بكامل بياناتها: اسم النشاط، المكان، التواريخ، العدد المتوقع
- رفع ملف الموافقة الإدارية كمرفق للدورة
- **توليد رابط فريد** لكل دورة يُشارَك مع المشاركين للتسجيل
- تتبع حالة الدورة: `PUBLISHED` → `CLOSED`
- عرض عدد المشاركين المسجّلين لكل دورة في الوقت الفعلي

### ب) نموذج تسجيل المشاركين

نموذج متعدد الخطوات يعمل على جميع الأجهزة:

**الخطوة 1 — البيانات الشخصية:**
- مسح جواز السفر تلقائياً (OCR) — يستخرج الاسم ورقم الجواز وتاريخ الميلاد وتاريخ الانتهاء
- تحقق فوري لكل حقل أثناء الكتابة
- الحقول: الاسم الكامل، رقم الجواز، تاريخ انتهاء الجواز، رقم الهوية، رقم الجوال

**الخطوة 2 — معلومات إضافية:**
- تاريخ الميلاد مع منتقي ذكي وأزرار سريعة (25، 30، 40 سنة)
- رقم الآيبان البنكي مع تحقق صيغة SA + 22 رقم

**الخطوة 3 — المرفقات:**
- صورة جواز السفر — تظهر معاينة مصغرة إذا مُسح مسبقاً
- صورة الهوية الوطنية
- يُقبل الصور فقط — رسالة إرشادية واضحة عند محاولة رفع PDF

### ج) مسح جواز السفر (OCR بدون اتصال خارجي)

1. يلتقط المشارك صورة للجواز بكاميرا جواله أو يختارها من المعرض
2. المتصفح يقصّ الجزء السفلي من الصورة (منطقة MRZ — 25% أسفل الصورة) ويعزّز التباين
3. `tesseract.js` يقرأ خطوط MRZ محلياً داخل الجهاز
4. يُحلَّل النص ويُستخرَج: الاسم الكامل، رقم الجواز، تاريخ الميلاد، تاريخ الانتهاء
5. تُملأ الحقول تلقائياً وتُعرَض للمراجعة والتعديل
6. **الصورة الأصلية لا تغادر جهاز المشارك** — حماية كاملة للخصوصية

### د) مرشحو جامعة نايف

ميزة لإضافة أفراد الكادر الجامعي المصاحبين للدورة (ليسوا مشاركين عاديين):
- إضافة مرشح: الاسم بالإنجليزي، رقم الجواز (اختياري)، الجوال (اختياري)، المسمى الوظيفي
- تعديل بيانات أي مرشح inline دون إعادة تحميل الصفحة
- حذف مع نافذة تأكيد لمنع الحذف العرضي
- تظهر قائمتهم في جدول مستقل **فوق** جدول المشاركين في قائمة PDF

### هـ) قائمة المشاركين PDF (الوثيقة الرسمية)

وثيقة PDF احترافية تُولَّد تلقائياً تحتوي على:
- **الترويسة:** شعار جامعة نايف (يمين) + "PARTICIPANTS LIST" (يسار)
- **بيانات الدورة:** الاسم، المكان، التواريخ، عدد المشاركين، المنسق
- **جدول مرشحي جامعة نايف** (NAIF UNIVERSITY NOMINEES) — إن وُجدوا
- **جدول المشاركين المسجّلين** (REGISTERED PARTICIPANTS): الرقم، الاسم، رقم الجواز، تاريخ الميلاد
- **التذييل:** Training Deputyship — NAUSS + تاريخ الإصدار + رقم الصفحة
- خط Cairo لدعم النصوص العربية

### و) إصدار التأمين

عند الضغط على "إصدار التأمين" وتأكيد العملية:
1. تُحذَف فوراً جميع ملفات المرفقات (صور الجوازات والهويات) من قاعدة البيانات
2. تُسجَّل العملية في سجل التدقيق مع الطابع الزمني واسم المستخدم
3. تُحدَّث حالة الدورة إلى `CLOSED`
4. يُولَّد رمز جديد للدورة للأرشفة
5. تُوضَّح حالة "تم إصدار التأمين" في واجهة القوائم

### ز) لوحة التحكم والتقارير

- إحصاءات فورية: الدورات النشطة، إجمالي المشاركين، مشاركو هذا الشهر
- قائمة الدورات الأخيرة مع حالة كل منها
- للمدير: رؤية شاملة لجميع الموظفين وإنتاجيتهم
- تبديل الدور (مدير ↔ موظف) بدون تسجيل خروج

## 12. دليل المشغّل اليومي

### إنشاء دورة جديدة

```
1. تسجيل الدخول ← الدورات ← + دورة جديدة
2. تعبئة: اسم النشاط، المكان، التواريخ، العدد المتوقع
3. رفع ملف الموافقة الإدارية
4. حفظ الدورة
5. نسخ "رابط النموذج" ومشاركته مع المشاركين
```

### إضافة مرشحي جامعة نايف

```
1. قائمة المشاركين ← اضغط على الدورة المطلوبة
2. في النافذة المنبثقة: أضف كل مرشح (الاسم، الجواز، الجوال، المسمى)
3. زر "تعديل" لتعديل أي مرشح لاحقاً
4. زر "حذف" مع تأكيد لحذف أي مرشح
```

### تصدير قائمة المشاركين

```
1. قائمة المشاركين ← اضغط "تصدير PDF" بجانب الدورة
2. يُولَّد PDF ويُفتح في تبويب جديد جاهزاً للطباعة
```

### إصدار التأمين

```
1. التأمين ← اختر الدورة
2. راجع قائمة المشاركين وتأكد من اكتمالها
3. اضغط "إصدار التأمين" ← ستظهر نافذة تأكيد
4. بعد التأكيد: تُحذَف الملفات وتُغلَق الدورة تلقائياً
```

### إدارة المستخدمين (المدير فقط)

```
1. القائمة الجانبية ← الإدارة ← المستخدمون
2. إضافة / تعديل / تعطيل / تغيير كلمة مرور أي موظف
```
